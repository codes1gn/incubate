The Pipeline

  • Phase 0
    Memory Load & Setup
    Load JSONL memory, detect platform, generate slug
  • Phase 1
    PRD
    8-section Product Requirements Document — no user input
  • Phase 2
    Self-Review
    3-round internal review: Completeness → Feasibility → Autonomy
  • Phase 3
    Architecture
    7-section design with Decision Log for best-of-N comparison
  • Phase 4
    Development
    Code with self-test checklist; no secrets, all MVP features
  • Phase 5
    Ship
    Push to GitHub via MCP tools (Copilot) or git CLI (Cursor)
  • Phase 6
    Test Framework
    13 pattern-based scenarios, smoke test must pass
  • Phase 7
    Batch Testing
    ≥1000 checks at ≥95% pass rate; auto-retry on failure
  • Phase 8
    README
    11-section README with badges, stats, platform table
  • Phase 9
    Website
    Dark terminal GitHub Pages + CI/CD workflows
  • Phase 10
    Verify
    Status report; note manual Pages activation requirement
  • Phase 11
    Memory Save
    Append ≥3 JSONL entries for future incubations
